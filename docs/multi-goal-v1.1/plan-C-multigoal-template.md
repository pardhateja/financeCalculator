Replace with:

```js
/* India FIRE example template — Pardha v1.1 (10 overlapping phases).
 * Decisions locked in docs/multi-goal-v1.1/00-tone-and-locked-decisions.md §3.1
 * Math engine sums overlapping phases per year (verified by qa-003 in v1). */
RP._phaseExampleTemplate = [
    // Always-on baseline
    { name: 'Base (no kids)',          startAge: 27, endAge: 100, baseMonthlyExpense: 50000,  inflationRate: 6  },
    // Kid 1: born when Pardha is ~28; at home until college (age 17 of kid → Pardha 45)
    { name: 'Kid 1 at home',           startAge: 28, endAge: 45,  baseMonthlyExpense: 10000,  inflationRate: 6  },

    { name: 'Kid 1 pocket money',      startAge: 45, endAge: 49,  baseMonthlyExpense: 15000,  inflationRate: 6  },
    // Kid 2: born when Pardha is ~35
    { name: 'Kid 2 at home',           startAge: 35, endAge: 52,  baseMonthlyExpense: 10000,  inflationRate: 6  },
    { name: 'Kid 2 college fees',      startAge: 52, endAge: 56,  baseMonthlyExpense: 8333,   inflationRate: 10 },
    { name: 'Kid 2 hostel',            startAge: 52, endAge: 56,  baseMonthlyExpense: 8333,   inflationRate: 7  },
```

**Note on Phase 1 startAge**: hardcoded to `27` (Pardha's current age at v1.1 spec time). This is fine because:
- Feature A (DOB → auto age) will replace it dynamically when Load Example reads `RP.val('currentAge')` to compute the Base startAge
- For v1.1 ship-now, 27 is a reasonable default that matches the existing Basics defaults

**Note on Phase 1 endAge**: 100 (= default lifeExpectancy). Same logic — Feature A could read `lifeExpectancy` dynamically. For v1.1 first cut, 100 is the safe baseline.

## Risks

- **10 cards crowd the UI**: design lead in v1 reviewed up to 4 phases. With 10, vertical scroll on mobile gets long. Acceptable for v1.1 — if Pardha finds it unwieldy in real use, defer to a "collapsed/expanded" card design in v1.2.
- **Allocation table with 10 rows**: similarly scrollable but matches existing `.projection-table` overflow pattern. OK.
- **Color palette rotates after 6 phases** — phase 7 (Kid 2 college fees) is blue again, same as Base. Causes mild visual confusion. Acceptable for v1.1.
