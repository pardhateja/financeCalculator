## JavaScript wiring

In `js/calc-income.js` (or wherever Basics tab init happens):

```js
// DOB → age compute helper
function computeAgeFromDOB(dobString) {
    if (!dobString) return null;
    const dob = new Date(dobString);
    if (isNaN(dob.getTime())) return null;
    const now = new Date();
    let age = now.getFullYear() - dob.getFullYear();
    const monthDelta = now.getMonth() - dob.getMonth();
    if (monthDelta < 0 || (monthDelta === 0 && now.getDate() < dob.getDate())) {
        age--;
    }
    return age;
}

function updateAgeFromDOB() {
    const dobEl = document.getElementById('dateOfBirth');
    const ageEl = document.getElementById('currentAge');
    const overrideEl = document.getElementById('currentAgeOverride');
    if (overrideEl && overrideEl.checked) return; // override active, don't touch
    const computed = computeAgeFromDOB(dobEl.value);
    if (computed !== null && ageEl) {
        ageEl.value = computed;
        // Trigger any downstream listeners (Projections, etc.) that watch for currentAge changes
        ageEl.dispatchEvent(new Event('change', { bubbles: true }));
    }
}

function setupDOBOverride() {
    const dobEl = document.getElementById('dateOfBirth');
    const ageEl = document.getElementById('currentAge');
    const overrideEl = document.getElementById('currentAgeOverride');
    if (dobEl) {
        dobEl.addEventListener('change', updateAgeFromDOB);
        // Initial compute on load
        updateAgeFromDOB();
    }
    if (overrideEl && ageEl) {
        overrideEl.addEventListener('change', () => {
            if (overrideEl.checked) {
                ageEl.removeAttribute('readonly');
            } else {
                ageEl.setAttribute('readonly', '');
                updateAgeFromDOB();
            }
        });
    }
}

// Hook into existing init pattern (look at how other Basics fields are wired)
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', setupDOBOverride);
} else {
    setupDOBOverride();
}
```

## Risks

- **Date input localization**: HTML `<input type="date">` shows in OS-locale format (e.g., DD/MM/YYYY in India, MM/DD/YYYY in US). Browser handles this. Stored ISO format (YYYY-MM-DD). Should be invisible to user.
- **Timezone edge case**: someone born on a date in a different TZ could see off-by-one age in the boundary day. Acceptable for v1.1 — users don't usually care about exact birthday boundary.
- **Override toggle persistence weird edge**: if user enables override + types age 35, then later changes DOB → DOB change is ignored (override still active). Pardha needs to uncheck Override for DOB to take effect. This is correct behavior (override means "user wins") but worth noting in the toggle's tooltip text: "Override: ignore DOB and type age manually".
