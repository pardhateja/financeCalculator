# Plan: Expense Profile dashboard (today's rupees, per-phase breakdown)

**Goal**: add a new section in Multi-Goal tab showing what monthly expenses look like at each age in TODAY's rupees (no inflation compounded), with per-phase breakdown. Locked in `00-tone-and-locked-decisions.md` §3.2.

## Why
With 10 overlapping phases, the inflated chart shows compounding growth that hides the real-life shape. This view shows the stair-step profile (kids → college → empty → medical) in real terms — useful for the human question "what's my actual monthly burn at age 45?"

## Data shape

`_buildExpenseProfileRows(phases, currentAge, lifeExpectancy)` returns:

```js
[
  {
    age: 27,
    activePhases: [
      { phaseId: '...', name: 'Base (no kids)', color: 'blue', monthlyToday: 50000 }
    ],
    totalMonthlyToday: 50000
  },
  {
    age: 45,
    activePhases: [
      { phaseId: '...', name: 'Base (no kids)',     color: 'blue',    monthlyToday: 50000 },
      { phaseId: '...', name: 'Kid 1 college fees', color: 'amber',   monthlyToday: 8333  },
      { phaseId: '...', name: 'Kid 1 hostel',       color: 'purple',  monthlyToday: 8333  },
      { phaseId: '...', name: 'Kid 1 pocket money', color: 'teal',    monthlyToday: 15000 }
    ],
    totalMonthlyToday: 81666
  },
  ...
]
```

For each age in `[currentAge .. lifeExpectancy]`:
- Find all phases active at that age (`startAge <= age <= endAge`)
- For each, record `monthlyToday = phase.baseMonthlyExpense` (NO inflation factor)
- Total = sum of monthlyToday across active phases

## Wiring

Chain into the existing render cascade so any phase mutation refreshes the new view too:

In `js/calc-multigoal.js`, the existing `wrapRenderPhasesForAllocation` IIFE (~line 1583) calls `renderAllocation` then `renderProjection`. Add a third call: `renderExpenseProfile`. Also add to the gap-banner wrapper (~line 1692) for consistency in case of wrap-order changes.

## Chart rendering

Reuse the existing Canvas 2D pattern from `RP.renderMultiGoalChart` in `js/chart.js`:
- X axis: age (currentAge → lifeExpectancy)
- Y axis: total monthly today's-rupees expense
- Phase-shaded vertical regions: same colors + tints as the corpus projection chart
- Single line (not stacked) showing the total

Or simpler: stacked area chart per phase. Decision: **single line for v1.1** (less Canvas code; per-phase detail lives in the table). Stacked area can be a v1.2 nice-to-have.

## Risks

- **74 table rows is long**. Consider grouping by 5-year buckets for the table (Age 27-31, 32-36, etc.) but year-precision is what Pardha asked for. Ship per-year for v1.1; defer grouping to v1.2 if it's noisy.
- **Active phases column gets wide** with 5+ active phases at peak overlap years. Use the existing `.phase-badge-group` wrapping pattern — already handles overflow.
- **Chart re-renders on every phase mutation** + may need debouncing if the user is typing in the form. Same risk as existing `renderProjection` chart — reuse the same pattern, no new debounce needed.
